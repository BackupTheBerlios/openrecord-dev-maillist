Yup, the shrinking and expanding is just plain wrong.  But I'm reluctant to
fix that right now...  

We still haven't thought much about what sort of UI components we want to
have within each table cell.  I think in the long run we're going to want to
have a UI that lets people add more than one entry into a table cell, and a
UI for selecting entries, deleting entries, reordering entries, etc.  Also a
UI for doing data type validation, and a UI for auto completion as you're
typing.  And a UI for entering references to other items by selecting them
from pull-down lists, or via drag-and-drop.  

All of which means that code for what's going on within a table cell is
likely to change a great deal in next few months, so it might not be worth
spending much time debugging the visual glitch in the current code.
