Hey Chao,

I switched all our code over to use the new data model, and everything seems
to work fine!

I wired up your sign-in/sign-out code so that it calls the login/logout
methods that the new data model offers.  I did a quick-and-dirty job of
that, so we'll probably want to revisit it at some point.  Next time we meet
up I can explain what I did and what I didn't do, and we can talk about how
we really want it to work.  

The data model now supports password-checking authentication, so we can add
UI for that whenever we want to.  The password storage isn't yet the least
bit secure, but I figured it was a step in the right direction to at least
have a functional API for authentication.

I also added a new method to the data model, so that a view can create a
"provisional item".  I new provisional items are intended to meet the need
for creating "blank items" or "template items" when the user is entering a
brand new row in a table.
