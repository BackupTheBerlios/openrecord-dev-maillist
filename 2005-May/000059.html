Wow, that TiddlyWiki thing is doing something very interesting.  I had seen
TiddlyWiki before, but I hadn't bothered to spend more than a few minutes
looking at it.  This time I spent a little more time, and I'm very glad I
did.

TiddlyWiki is designed to run on your local machine, running from a
"file://" URL rather than an "http://" URL, so that you don't have to have
Apache running on your local machine.  

Whenever you make a change to a wiki entry, the TiddlyWiki JavaScript code
saves your new data file to the local file system.  I actually had thought
that was impossible.  I thought JavaScript had no mechanism for saving data
to a file on the local machine.  

Sometime when we have time, it'd be great to study the TiddlyWiki code that
does local saves, and understand the mechanism they're using.  It involves a
bunch of JavaScript components I know nothing about: the "netscape" object's
security PrivilegeManager, and UniversalXPConnect, and
Components.interfaces.nsILocalFile and
Components.interfaces.nsIFileOutputStream.  Looks like it might all be a
little brittle, but it'd be wonderful if OpenRecord could support the
entirely new use case of having someone run OpenRecord on their home machine
to manage personal data!
