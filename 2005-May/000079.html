A few more questions/comments for discussion after working on the code:
1) Should there be a Query class? Maybe also Attribute?
2) Should we create a Plugin base abstract class? I think this will  
be helpful, and will help refactoring all the other plugins to not  
depend on SectionView.getListOfContentItems()
3) We need to discuss the usage and UI for Auto Completion (i.e.  
SuggestionBox):
     - we need to add arrow key and tab/return support
         - implementation issue: what's the easiest way to implement  
selection of SuggestionBox item?
     - should only certain attributes do "Auto Completion" by default  
and not others? if so, what criteria do we use to decide?
     - when doing auto-completion for attributes, should we can  
through all entries for this attribute? or just all entries within a  
certain context e.g. all entries for attribute X in Table Y?

4) In model/StubVirtualServer.js Line 193, I'm making a call to a  
protected method, can you see what's the best way to fix this problem?

5) How do you do Util.assert(x != null)? Util.assert(x) doesn't pass  
the LintTest

6) Should we consider not supporting "strict line ending" in the  
LintTest? the ";" at the end of prototype functions is really getting  
to me.
