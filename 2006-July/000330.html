check out this really cool open source javascript timeline widget:

http://simile.mit.edu/timeline/

It uses a simple XML format as input so it seems like it could be
integrated in openrecord relatively easily (and its part of the Simile
RDF project at MIT, so hopefully they're thinking about how to
represent time series in RDF)

While I'm at it, I recently stumbled across some open source pivot
table and OLAP apps. If OpenRecord ever takes on Wikipedia size data
sets (e.g. a WikiAlmanac) it will probably have to go down the OLAP
route, so it's nice to know there are some serious open source options
out there.

http://jpivot.sourceforge.net/index.html

its made by
http://www.jasperreports.com/
which is trying to to be an open source OLAP/Business Intelligence vendor.

(see also http://en.wikipedia.org/wiki/Open_Source_BI )

And here's another open source web app that uses jpivot to navigate data cubes:
http://rubik.sourceforge.net/

-- adam

On 7/11/06, Brian Douglas Skinner <skinner at dojotoolkit.org> wrote:
> Here's a great talk by Hans Roslings, from the 2006 TED conference, if
> you've got Flash installed, and a broadband connection:
>    http://www.ted.com/tedtalks/tedtalksplayer.cfm?key=hans_rosling
>
> Hans Rosling is a public health expert, and the talk is nominally about
> global development, but the talk also serves a great example of what can
> be done with the visual display of quantitative information.  In
> particular, Rosling makes a lot of use of bubble plots, which is
> especially topical for OpenRecord, since Hiran is working this week on a
> scatter-plot/bubble-plot plugin for OpenRecord.
>
> The whole talk is about 20 minutes long.  Watching just the first 5
> minutes, that's enough to see some of the cool plots.  If you watch the
> whole thing, in the last 5 or 6 minutes he talks about data
> visualization software, and accessing database over the Internet.
>
> Rosling shows a bunch of plots that are animated, displaying 40 years of
> time-series data in a 30-second animation.  We've talked about having
> animated views in OpenRecord too, and there's no reason we can't do that
> (just a small matter of programming).
>
> Rosling's talk is also a nice reminder about the importance of time
> series data.  The current OpenRecord data model doesn't have any sort of
> intrinsic support for time-series data, nor does the experimental
> dojo.data code that I wrote earlier this year, nor does the data model
> code that IBM is contributing to dojo.  It would be cool to have time as
> a first-class notion in the data model, but maybe that's too ambitious.
>   I'm known for undertaking efforts that are too ambitious!
>
> One last cool thing -- Rosling's talk is released under a Creative
> Commons license.  I'm psyched to see the CC licenses becoming more and
> more common.
>
